Web开发总结(一年)

发表于 2017-02-12 | 分类于 随记 |

最近看到学校某学院网站的源码，深吸一口气，怎么可以这样写？所有的逻辑都写在了PHP页面上，甚至包括连接数据库的参数。是不是让我写就会好很多呢？打开了之前某机场售票项目代码，那是我一年前上线的第一个独立负责的项目，看了几眼其中的代码，顿时也哭笑不得。喜的是，一年过去了，能知道过去的不足，说明这一年进步了。要知道当时这个项目上线时，第一天就产生400多交易订单，那会对自己的代码多么自信。忧的是，当时尽管已经有意识从包管理、类命名、变量命名等角度去规范代码，但还停留在人云亦云的角度，比如将大量的业务逻辑写在了Controller里面，而这原本应该是很薄的类。这带来的直接坏处就是，后面开发其他模块时，没法直接复用业务代码，本质上就是还没有意识到抽象出Service层。是不是抽象出Service层就好了？带着这些问题，用了几天时间陆续整理出这篇总结。

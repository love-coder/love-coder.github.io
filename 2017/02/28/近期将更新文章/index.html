近期将更新文章

发表于 2017-02-28 | 分类于 随记 |

写完这几篇技术文章之后发现，写文章的难度远超过我当初的想象。不仅仅要把问题说明白，同时为了和其他类似文章有所区别，还需要研究下底层原理，好融会贯通，沉淀基础。而光是将问题说明白难度已不小，这需要能够比较全面的看待问题，并给出切实可行的解决方案。同时，每天都有新技术出现，若一直追寻这些技术，则容易陷入各种困境。可喜的是这些新技术的底层原理是相似的，这就是为什么除了把事情说明白之外，还要在必要时深入去了解底层实现。这样，在下次遇到新的技术时，可以类比学习，触类旁通，并保持对技术的热情。

《Web开发总结(一年)》
这篇文章还有后续Servlet规范、Web框架、业务思考等需要完成。

《Linux服务器排错心得》
纯碎会在Linux上部署和简单操作，并不能满足对后台开发人员的要求。必须深入了解相关底层原理，并在此基础上形成一套行之有效的排错思路。

《SIMIPN重构中的一些思考》
早期SIMIPN只有我一个人开发和维护，现在加入3位新同学以及未来更多同学将加入，必须对原来代码做一次全面的重构。这篇文章记录了重构过程中遇到的种种问题，以及我的解决思路。

《动态规划》
动态规划算法是最基础的算法之一，不仅仅在各种面试中被频繁问起，而且在日常项目开发中也有出现。

《基于TCP/UDP的网络开发》
不管是什么项目开发，都难免跟底层通信打交道，不了解这些协议常见的一些知识点，对于一些异常就很难有深刻的理解。

欢迎讨论!

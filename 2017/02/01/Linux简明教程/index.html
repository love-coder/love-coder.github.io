Linux 简明教程

发表于 2017-02-01 | 分类于 操作系统 |

最近工作中一些几乎没有Linux基础的同学在开发过程中遇到一些问题，所以我花了两天时间整理了这些基础知识供其查找，以满足日常工作需要。文章分为四个部分：Linux目录结构、常用软件安装方法、vim基本使用，并在最后整理了30条左右的常用指令。因为定位是简明教程，所以不会涉及过多原理以及对每条指令展开阐述。该文章均是以64位CentOS 7.1为例，其他发行版也基本类似。

Linux目录结构

Linux系统中所有资源均是以文件的方式被表示。常见的有：普通文件、目录文件、字符设备文件和块设备文件、符号链接文件等。一般认为Linux是没有扩展名的概念，也就是说一个文件能不能被执行跟其扩张名没有一点关系，和在Windows平台下的.exe、.bat等不同。只要用户账户具备x即执行权限，就可以执行该文件，当然能不能执行成功就得看具体内容。但是，为了能够更好地理解Linux，开源社区任然建议遵循约定俗成的一些规范，比如.sh是脚本或批量处理文件，.tar.gz、.tar、.zip等是压缩文件，.html是网页文件等等。同样，被约定俗成的还有它的目录结构，它的各部分含义如下：

而 / 根目录是系统最重要的目录，不仅仅是因为上述目录均是由其衍生而来，而且它还包含系统开机和关机等操作的文件和函数库文件。因此，应用程序安装的软件最好不要与根目录在同一分区，且保持根目录越小越好，这样不易出现问题。

su – Swtich User

格式：su [选项] [参数]

功能：Linux较于Windows很大的一点好处就是增强了对于权限的管理，比如有些操作只开放给一些特定的用户，如果不具备该权限的账号想完成该操作，就需要切换到其他具有该权限的账户。超级用户拥有绝对权限，即切换到其他用户不需要输入密码。

常用参数：

  • 或 -l 改变身份的同时，也更改工作目录、Shell环境以及path变量等
    -c 执行完指令后，恢复原来身份

chmod

格式：chmod [选项] [参数]

功能：Linux所有资源均被表示成文件的形式，而chmod指令即用来修改文件或目录的访问权限。权限分为r（读）、w（写）和x（执行）三类，用户也分为三类为文件创建者、同组用户和其他用户。比如下面test.txt，其中-rw-r–r–，共9个字符每三个分别代表一种用户，它们表示文件所有者rw-，同组用户和其他用户r–，即文件所有者可以读和写文件但不可以执行。文件所有者可以赋予所有其他所有用户，但其他权限需要超级管理员或相应管理员或赋予。为了方便，一般用数字代表相应的权限，r在最高位用4表示，w在中间用2表示，x在最低位用1表示，比如7就代表所有用户均具备所有权限。

常用参数：

-R递归赋值目标目录所有字文件的权限

kill 和killall

格式：kill 或killall[选项] [进程]

功能：终止指定进程。一般终止前台进程可以使用Ctrl+C，而后台进程就需要这两个指令。通过ps、top等指令获得PID，然后借助kill就可以杀掉该进程。默认是发送编号为15的TERM信号，但对于一些无法捕获该信号的进程一般发送编号为9的信号，强行杀掉。不到外不得已一般不使用，因为它会导致正在运行中的数据丢失。而kilall指令是用来杀掉指定名称的进程。root用户使用将影响所有用户进程，而非root用户只对影响自己进程。

kill常用参数：

-l 列出所有信号
-s 发送指定信号
-p 只打印相关进程好，而不发送信号
-u 指定用户进程

killall常用参数：

-Z 只杀死拥有scontext 的进程
-e 要求匹配进程名称
-g 杀死进程组而不是进程
-i 交互模式，杀死进程前先询问用户
-w 等待进程死亡

功能：显示显示系统使用和空闲的内存情况，包括物理内存、交互区内存(swap)和内核缓冲区内存（共享内存将被忽略）。
常用参数：

-b 　以Byte为单位显示内存使用情况。
-k 　以KB为单位显示内存使用情况。
-m 　以MB为单位显示内存使用情况。
-g 以GB为单位显示内存使用情况。
-o 　不显示缓冲区调节列。
-s<间隔秒数> 　持续观察内存使用状况。
-t 　显示内存总和列。

借助GitHub Pages搭建博客

发表于 2017-01-27 | 分类于 随记 |

GitHub Pages免费提供300M不限流量的空间，提供给全球用户搭建公共静态页面托管服务。我看到网上不少人担忧这样好的服务如果被国人滥用，可能会导致GitHub Pages封锁国内账号，从而带来更大的不便。希望大家一起严于律己，更好的利用这个资源，让更多的技术人因此受益。

这篇文章记录了利用GitHub Pages搭建个人博客的所有过程。文章分为三个部分：快速开始、变得更好和其他补充。快速开始部分，我将仅通过Git工具和一个最简单的index.html文件快速搭建一个demo，让大家尽快了解GitHub Pages搭建个人博客的思路和原理；变得更好部分将借助基于Node.js的Hexo博客框架更加方便的来写博文；最后在其他补充部分将介绍搭建过程中遇到的一些问题，比如如何绑定域名、搭建图床等。

我希望您已经有Git和Node.js的一些基础。如果没有，也没有关系，遇到了可以先试着跟着操作，后面再去学习。

tips：建议主页显示文章的摘要，不然NexT默认主页显示是文章的全部内容，这对于有些很长的文章来说非常不美观。您可以在文章中的配置以下，通过添加区分摘要。具体而言，文章配置以下，以上为摘要用来在主页中预览，而以下，正常Markdown书写文章。点开阅读全文后，摘要会出现在文章首部。

3. 添加其他插件

2. 绑定域名

（1）申请域名

为了可以有一个好记的域名，而不必使用love-coder.github.io类似默认域名，我们可以在阿里云、腾讯云等处购买自己的域名，比如我的域名hanchun.tech是在阿里云上购买，五年共67元，很方便也很便宜。购买完成后，实名认证并且进行DNS创建和域名解析。阿里云默认在万网上创建，当然如果选择其他DNS，比如在DNSpod上，同样可以做相应修改。

3. 搭建图床

考虑到GitHub Pages存储空间有限以及图片加载慢等问题，我们借助七牛云存储实现图片存储。得益于开源世界的贡献，Hexo可以很方便的通过同步插件实现图片上传和页面显示，它的使用方法参考插件hexo-qiniu-sync的说明。

这里只做几点说明：

（1）配置参数urlPrefix，七牛做了调整，不能简单按照原文档说明方法去配置。具体方法，在七牛开发中心中的存储中心选择对应空间后，要不直接选择测试域名（虽说测试域名，但也能满足博客访问需求）或者新建加速域名。前者操作简单，直接负责粘贴，后者需要绑定的域名是必须经过大陆备案。我用的.tech域名，目前还没资格备案，呵呵，所有选择的第一种方法。

（2）七牛提供了非常多的图片处理工具，只需相应的加上即可。这里可以参考开发者中心中的图片处理API文档。但一般言，了解其中基本处理即可，比如imageView2/2/w/500/h/335代表限定缩略图宽最多500像素，高最多335像素，等比缩略。
